# Contributing to laravel-aigeo

First off, thank you for considering contributing to `laravel-aigeo`! It's people like you that make `laravel-aigeo` such a great tool.

## Contributing Workflow

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally.
3. **Create a new branch** for your feature or bug fix.
4. **Make your changes**.
5. **Run tests** using `./vendor/bin/pest`.
6. **Commit your changes** with descriptive commit messages.
7. **Push your branch** to your fork.
8. **Submit a pull request** to the main repository.

## Code Style

Please use [Laravel Pint](https://laravel.com/docs/master/pint) to maintain a consistent code style:
```bash
./vendor/bin/pint
```

## Testing Requirements

All pull requests must include tests. We use [Pest](https://pestphp.com/) for our test suite.

## Branch Naming

- `feature/description` for new features
- `fix/description` for bug fixes
- `docs/description` for documentation updates
